Command line scanner

Run a full security scan on any skill file, directory, or MCP manifest. Text, JSON, and SARIF output for every workflow.

  • Pattern, YARA, Semgrep, and LLM engines in one command
  • OWASP MCP Top 10 mapping on every finding
  • Toxic flow detection across all findings
  • Exit code 2 on HIGH+ for CI/CD
terminal
$ bawbel scan ./skills/ --recursive
Bawbel Scanner v1.1.0
🔴 CRITICAL AVE-2026-00001
External instruction fetch detected
OWASP MCP: MCP04, MCP06
⛓ CRITICAL RCE Chain 9.7
Risk: 9.7/10 Findings: 2 Toxic flows: 1

VS Code extension

Inline diagnostics as you write. Squiggles on every finding with hover details, severity, and fix suggestions. Auto-scans on save in under 25ms.

  • Auto-install CLI on first activation, zero setup
  • Right-click to suppress false positives inline
  • Full scan report with Cmd+Alt+R
  • Status bar shows live finding count

Pre-commit hook

Block malicious skills at the commit boundary. Installs bawbel-scanner automatically in an isolated virtualenv. No manual pip install needed.

  • Two hooks: bawbel-scan and bawbel-scan-all
  • Blocks commit with remediation message
  • Local mode for air-gapped environments
  • Configurable severity threshold
git commit
$ git commit -m "add search skill"
Bawbel Scanner..............Failed
AVE vulnerabilities found (HIGH+):
[HIGH] AVE-2026-00004 skill.md line 7
Run: bawbel report skill.md

CI/CD integration

One action for GitHub. SARIF upload to the Security tab. Findings appear as inline PR annotations. Blocks merges on HIGH+ automatically.

  • GitHub Actions, GitLab CI, Jenkins, CircleCI
  • Azure DevOps and Bitbucket Pipelines
  • SARIF 2.1.0 output for all platforms
  • Combined scan and pin check in one step
.github/workflows/bawbel.yml
- uses: bawbel/bawbel-integrations@v1
with:
path: .
fail-on-severity: high
format: sarif
3 findings uploaded to Security tab

MCP server scanning

Fetch and scan server-cards before your agent connects. Injections at the discovery layer fire before any tool call is made.

  • Fetches .well-known/mcp.json automatically
  • Scans tool descriptions and parameter schemas
  • Same JSON/SARIF output as bawbel scan
  • Conformance scoring with bawbel conform
$ bawbel ssc https://api.example.com
Fetching .well-known/mcp.json...
🔴 CRITICAL AVE-2026-00041
MCP Server-Card Injection
$ bawbel conform https://api.example.com
Score: 87/100 Grade: B Conformant: yes

Rug pull detection

Hash skill files and detect content drift. Pins committed to git so changes show in code review and the whole team shares them.

  • SHA-256 hashes with pinned_by audit trail
  • fail-on-drift for CI exit code 2
  • Beats Snyk: git-visible vs local ~/.mcp-scan
  • Works on any machine after git clone
$ bawbel pin ./skills/
Pinned 4 files to .bawbel-pins.json
$ bawbel cp ./skills/ --fail-on-drift
1 file drifted from pin
skills/search.md hash mismatch
Pinned: abc123 Current: 999evil

Watch mode

Bawbel watches your skill directory and re-scans instantly on every file change. See results in real time as you write.

  • Re-scans in under 25ms on file save
  • Works with any editor, not just VS Code
  • Ctrl+C to stop, all results logged to terminal
  • Install with pip install "bawbel-scanner[watch]"
watch mode
$ bawbel scan ./skills/ --watch
